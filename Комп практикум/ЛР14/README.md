# Лабораторная работа №14

## Веб-скрапинг сотрудников РГПУ им. А. И. Герцена

# Цель работы

Целью лабораторной работы являлось изучение основ веб-скрапинга в Python, получение практических навыков работы с HTML-документами, HTTP-запросами и автоматическим сбором данных с веб-сайтов.

В рамках работы необходимо было реализовать сбор данных сотрудников РГПУ им. А. И. Герцена двумя способами:

1. С использованием библиотеки BeautifulSoup
2. С использованием фреймворка Scrapy

---

# Постановка задачи

Необходимо было выполнить веб-скрапинг сайта:

[https://atlas.herzen.spb.ru/](https://atlas.herzen.spb.ru/)

Требовалось:

* пройти по страницам преподавателей с 1 по 54,
* получить список сотрудников,
* извлечь ФИО и ссылку на профиль,
* перейти на страницу профиля каждого преподавателя,
* получить email и номер телефона,
* сохранить полученные данные в CSV-файл.

# Реализация с использованием BeautifulSoup

В первой части работы был реализован веб-скрапер с использованием библиотеки BeautifulSoup.

Алгоритм работы программы:

1. Выполнялся HTTP-запрос к страницам преподавателей
2. Получался HTML-код страницы
3. Из HTML извлекалась таблица преподавателей
4. Выполнялся проход по строкам таблицы
5. Получались:

   * ФИО
   * ссылка на профиль
   * должность
   * факультет
   * кафедра
   * учёная степень
6. Выполнялся переход в профиль преподавателя
7. Из профиля извлекались:

   * email
   * номер телефона
8. Все данные сохранялись в список
9. На основе списка формировался CSV-файл

Для анализа HTML использовались:

* find()
* find_all()

Для сохранения результата использовалась библиотека pandas.

---

# Реализация с использованием Scrapy

Во второй части работы был реализован аналогичный веб-скрапер с использованием фреймворка Scrapy.

В отличие от BeautifulSoup, Scrapy представляет собой полноценный framework для профессионального веб-скрапинга.

Алгоритм работы:

1. Spider автоматически проходил по страницам преподавателей
2. Извлекались ссылки на профили сотрудников
3. Выполнялся переход на страницы профилей
4. Получались:

   * ФИО
   * должность
   * факультет
   * кафедра
   * учёная степень
   * email
   * номер телефона
5. Scrapy автоматически экспортировал данные в CSV-файл

В процессе работы использовались:

* CSS-селекторы
* XPath
* response.follow()
* yield

---

# Сравнение BeautifulSoup и Scrapy

| BeautifulSoup                   | Scrapy                                 |
|---------------------------------|----------------------------------------|
| Простая библиотека              | Полноценный framework                  |
| Легче изучать                   | Более сложная архитектура              |
| Подходит для небольших проектов | Подходит для крупных scraping-проектов |
| Ручная работа с requests        | Встроенная система HTTP-запросов       |
| Более низкая производительность | Более высокая производительность       |
| Нет встроенной асинхронности    | Поддержка асинхронности                |

BeautifulSoup оказался удобным для быстрого старта и небольших проектов.

Scrapy показал более профессиональный подход к организации веб-скрапинга и более высокую производительность.
