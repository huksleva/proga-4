# ОТЧЁТ по выполнению ЛР №10
**Тема:** Веб-приложение для генерации изображений с использованием модели `black-forest-labs/FLUX.1-schnell` и фреймворка Flask

---

## 1. Цель работы
Разработать и протестировать веб-приложение на Python, предоставляющее REST-маршруты `/login` и `/makeimage`. Приложение должно принимать пользовательские параметры (ширину, высоту, текстовый запрос), выполнять строгую валидацию, обращаться к API Hugging Face для генерации изображения, обрабатывать результат с помощью библиотеки Pillow и возвращать готовый файл в формате JPEG. Особое внимание уделено безопасному хранению учётных данных, обработке ошибок и соответствию техническому заданию.

## 4. Ход реализации

### 4.1. Маршруты приложения
- `GET /login` → возвращает JSON-объект `{"author": "ваш_логин_в_moodle"}`.
- `GET /makeimage` → отдаёт HTML-страницу с формой (`width`, `height`, `text`). При ошибках валидации или генерации шаблон получает переменную `message` для отображения предупреждения.
- `POST /makeimage` → принимает `application/x-www-form-urlencoded`, валидирует данные, вызывает модель, возвращает бинарные данные `image/jpeg`.

### 4.2. Валидация входных данных
Реализована строгая проверка согласно ТЗ:
1. `width` и `height` приводятся к `int`. При `ValueError` → `"Invalid image size"`.
2. Проверяется положительность значений.
3. Проверяется кратность 32 (`% 32 != 0`) → `"Width and height must be multiples of 32"`.
4. Ограничение диапазона `256–1024` пикселей для снижения нагрузки на API и гарантии качества генерации.

### 4.3. Генерация и обработка изображений
- Инициализация клиента: `InferenceClient(token=HF_TOKEN, timeout=30)`
- Вызов модели: `client.text_to_image(prompt, model="black-forest-labs/FLUX.1-schnell", width=w, height=h)`
- Пайплайн обработки Pillow:
  ```python
  buffer = io.BytesIO()
  img.save(buffer, format="PNG")
  image = Image.open(buffer).convert("RGB")
  if image.size != (w, h):
      image = image.resize((w, h), Image.Resampling.LANCZOS)
  jpeg_buffer = io.BytesIO()
  image.save(jpeg_buffer, format="JPEG", quality=90)
  ```
- Ответ клиенту: `send_file(jpeg_buffer, mimetype="image/jpeg")`

### 4.4. Обработка ошибок
Все исключения при вызове API (`Timeout`, `HTTPError`, `RateLimit`, `QuotaExceeded`) ловятся в блоке `except Exception as e`. Приложение возвращает ту же HTML-форму с сообщением вида `Model generation failed: <текст ошибки>`, сохраняя введённые пользователем данные.

### 4.5. Безопасность
Токен доступа к Hugging Face не хранится в исходном коде. Он загружается через `python-dotenv` из файла `.env`, который исключён из системы контроля версий через `.gitignore`.

## 5. Инструкция по запуску и тестированию
1. Создать окружение и установить зависимости:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
2. Создать файл `.env` и добавить токен:
   ```env
   HF_TOKEN=hf_ваш_реальный_токен
   ```
3. Запустить сервер:
   ```bash
   python app.py
   ```
4. Тестирование:
   - `http://localhost:5000/login` → проверка JSON-ответа.
   - `http://localhost:5000/makeimage` → отправка валидных и невалидных данных, проверка сообщений об ошибках, проверка возврата JPEG.

## 6. Использование LLM и промпт-инжиниринг в ходе разработки
В соответствии с методическими указаниями, при написании и отладке кода применялись следующие техники:
- **Chain of Thought (ручной режим):** Логика валидации и обработки изображения разбивалась на последовательные шаги (парсинг → проверка типа → проверка кратности → вызов API → конвертация Pillow → возврат ответа).
- **Автоматический CoT:** Запросы к LLM формулировались с явным указанием промежуточных этапов обработки ошибок и типов данных, что позволило сразу получить корректные конструкции `try/except` и работу с `io.BytesIO`.
- **Негативный промпт:** При тестировании генерации в `prompt` добавлялись конструкции типа `ugly, blurry, low quality, deformed, text, watermark`, что повысило стабильность качества выходных изображений. (Опционально можно вынести в отдельное поле формы при расширении функционала).
