Веб-приложение на Flask с использованием библиотеки Pillow

Разработка, контейнеризация и деплой облачного приложения

Flask Pillow Docker Gunicorn

Шаг 1: Постановка задачи

Создать веб-приложение на Flask, которое демонстрирует работу с библиотекой Pillow для обработки изображений и может быть развёрнуто в контейнере.

  • Реализовать маршрут /login, возвращающий JSON с автором.
  • Создать главную страницу с описанием этапов работы.
  • Подготовить Dockerfile для сборки образа.
  • Использовать gunicorn как production WSGI-сервер.

Шаг 2: Разработка приложения

Написано Flask-приложение с тремя маршрутами:

  • /login (GET) – возвращает JSON {"author": "ваш_логин"}.
  • / (GET) – отдаёт HTML-страницу (этот шаблон).
  • /api/process-image (POST) – генерирует изображение средствами Pillow и возвращает его в base64.

Библиотека Pillow используется для создания изображений «на лету».

Шаг 3: Контейнеризация

Создан Dockerfile на базе python:3.11-slim.

  • Установлены системные зависимости для Pillow.
  • Скопированы requirements.txt и исходный код.
  • Запуск через gunicorn на порту 8080.
  • Образ можно собрать командой: docker build -t flask-pillow-app .

Шаг 4: Деплой

Приложение готово к развёртыванию в облачной среде (Yandex Cloud, AWS, Heroku).

  • Используется лёгкий образ Python.
  • Порт 8080 экспонируется наружу.
  • Переменная окружения MOODLE_LOGIN позволяет задать автора.
